## Overview

I built this portfolio as a content-driven application where each project lives as a standalone MDX file. The objective was to keep updates frictionless while still delivering a premium interface and professional narrative.

## What I focused on

- Project content can evolve quickly without touching component logic.
- MDX gives me structured metadata and rich storytelling in one place.
- The architecture supports clean categorization by frontend, full stack, and AI.

## Technical details

- App Router implementation with Next.js and TypeScript.
- Reusable UI primitives and utility-first styling for consistency.
- Static project discovery by reading MDX files and parsing frontmatter metadata.
